Ubuntu Desktop 14.04 64位系统

下载源文件

推荐使用git命令，克隆源代码到本地。如没有安装git，请终端输入sudo apt-get install git

下载 CuraEngine源代码:
git clone https://github.com/Ultimaker/CuraEngine.git

下载libArcus源代码:
git clone https://github.com/Ultimaker/libArcus.git

下载protobuf源代码:
git cllone https://github.com/google/protobuf.git

PS:如果下载速度比较慢（可能墙），可以选择在上面的github的页面直接下载zip文件

编译环境配置

编译前需要先前搭建必备的开发环境

python3以上的python环境，包含SIP工具包:
sudo apt-get install python3-dev python3-sip python3-sip-dev

lsprotobuf相关工具包:
sudo apt-get install libtool autoconf gawk

cmake，gcc相关编译环境包:（以上编译环境如果使用apt安装，其安装版本过低。在编译安装过程会出现报错现象，这里介绍手动安装的方法）

cmake:
在https://cmake.org/files/v3.3 下选择 cmake-3.3.0-Linux-x86_64.tar.gz下载
终端读取到文件目录： sudo cp xx/cmake-3.3.0-Linux-x86_64.tar.gz /usr
解压文件；sudo tar zxvf cmake-3.3.0-Linux-x86_64.tar.gz
建立软连接： sudo ln -s /usr/cmake-3.3.0-Linux-x86_64/bin/* /usr/bin
终端测试：cmake –version 输出：cmake version 3.3.0为正常

gcc,g++:
在基础的apt源中gcc,g++版本为4.8.4，在编译中需要4.9.0版本以上
这里介绍将Ubuntu 14.04 的gcc/g++版本升级为4.9.X版本
首先在终端输入：
sudo add-apt-repository ppa:ubuntu-toolchain-r/test
sudo apt-get update
sudo apt-get install gcc-4.9 g++-4.9
测试版本：终端输入->gcc -v 若出现gcc version 4.9.X则成功，无需下面步骤。否则执行下面步骤
将4.9版本的gcc/g++设置为默认：
sudo update-alternatives –install /usr/bin/gcc gcc /usr/bin/gcc-4.9 60 –slave /usr/bin/g++ g++ /usr/bin/g++-4.9
再次终端输入：gcc -v测试

源码编译

Install Protobuf

cd protobuf
./autogen.sh(若报错：权限不够，请先sudo chmod +x autogen.sh)
./configure
make
sudo make install

Install LibArcus

cd libArus
mkdir build && cd build
cmake ..
make
sudo make install

Install CuraEngine

cd CuraEngine
mkdir build && cd build
cmake ..
make
